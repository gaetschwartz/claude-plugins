# Planning — course skeleton & section expansion

One module, two zoom levels. Both produce **outlines only** — titles, objectives, grammar points. Writing actual lesson content is `lesson-builder.md`'s job, done just-in-time.

## Level 1 — course skeleton (once per course)

1. **Inputs:** Course row + learner profile (goals, level, time budget, exam target).
2. **Research (2–4 web searches max):** the standard progression for the target language — the relevant framework (CEFR; TOPIK/JLPT/HSK/DELE… when an exam is a goal) and how well-known textbook series sequence topics. You want the *conventional order*, not content.
3. **Design:** 8–15 sections from the learner's starting point to their goal level (default: solid A2, extendable later). **Section 0 is mandatory when the target uses a script the learner doesn't know.** Theme sections functionally (introduce yourself, ordering food…), grammar woven in. Each ≈ 2–4 weeks at the learner's pace. Spiral: later sections revisit earlier ground harder.
4. **Confirm in chat** (numbered names + one-line objectives, instruction language), then **write in one batch**: Section rows — Name ("1 — …"), Course relation, Order, Status = Skeleton, Objectives = 3–5 can-do statements.
5. **Pointer:** Current Section → the Order-1 section for true beginners; for learners claiming prior knowledge, run `evaluation.md § Placement` now instead — it sets Current Section and Level.
6. **Log:** Type Planning, Next Step = "Expand section N".

## Level 2 — expand the current section (each time a section is reached)

Trigger: current section Status = Skeleton, or the learner asks to plan ahead.

1. **Read:** the Section row; the previous section's Eval Score; the latest Evaluation entry in the Session Log (weak points); optionally a Vocabulary query for high-Lapses words. **First section of a course: none of these exist — plan from the profile alone.**
2. **Adapt:** previous Eval Score < 70 → lesson-1 stub is a consolidation lesson targeting the logged weak points. Words with Lapses ≥ 3 get named in the stubs they should be recycled into. Respect the profile's pace.
3. **Design 5–10 lesson stubs.** A stub is: Name, Order, one-line objective, Grammar Points, vocabulary theme + rough word count, and any recycle/consolidation notes. **That's all** — no example sentences, no word lists.
4. **Confirm the lesson list in chat**, then batch-write Lesson rows: Name, Section + Course relations, Order, Status = Planned, Grammar Points, body = the stub.
5. **Flip state:** Section → Expanded; course Current Lesson → the first stub.
6. **Log:** Type Planning, Next Step = "Run lesson: <name>".

Both levels: confirm-then-write, everything in the instruction language.
